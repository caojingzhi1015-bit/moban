# 02_resume_parser — 个人简历解析模块

## ① 模块解决的核心痛点
- **图片/扫描简历无法提取**: 手机拍照或扫描版简历，OCR 后还需结构化，传统方式需人工逐字段录入
- **信息提取残缺**: 教育/工作/项目混在一起，AI 容易漏提、合并不相关内容
- **AI 幻觉编造经历**: 缺少溯源约束时，LLM 会自行补全公司名、职位、技能，生成虚假信息

## ② 完整执行业务流水线
```
用户上传简历 (图片/PDF/Word/文本)
  → OcrPdfProcessor 离线解析 (PaddleOCR / MinerU / pdfplumber)
  → 自动分章节 (教育/工作/项目/技能/证书)
  → 正则预提取硬字段 (姓名/电话/邮箱/城市) — AI 遗漏兜底
  → MultiModelGateway AI 结构化抽取 (temperature=0, 专属 Schema)
  → 合并结果 (硬字段覆盖 AI 遗漏项)
  → FactCheckValidator 双层校验 (格式 + 溯源)
  → 输出标准化简历 JSON (source_index 全程可追溯)
```

## ③ 防幻觉 / 防漏提取核心机制
- **正则硬字段兜底**: 手机号/邮箱等强格式字段用正则优先提取，AI 失败也不丢数据
- **分章节解析**: 先按章节标题拆分文本，避免不同段信息互相污染
- **source_index 溯源**: 每一条教育/工作/项目绑定原文行号，可逐条验证真实性
- **禁止占位符**: 全局 Prompt 强制拦截"从业从业者""掌握专业技能"等无效描述
- **temperature=0**: 关闭模型随机性，同一条简历多次解析结果一致

## ④ 与其他模块的数据互通逻辑
- **输出给 03_info_enquiry_agent**: 简历技能清单 → 对比 JD 发现缺口 → 生成追问
- **输出给 04_target_resume_generator**: 结构化简历 JSON → 按 JD 权重改写重组 → 生成简历
- **输出给 05_ai_interviewer**: 个人经历 → 面试官据此提问（不会问简历中不存在的内容）
- **输入来自 common/ocr_pdf_processor**: 所有文件解析统一入口
- **输入来自 common/multi_model_gateway**: AI 调用统一经过网关
