# 04_target_resume_generator — JD 对标简历生成 + Word/PDF 导出

## ① 模块解决的核心痛点
- **简历与JD不匹配**: 求职者用同一份简历投所有岗位，没有针对JD调整关键词和经历排序
- **缺乏中英文双版**: 外企需要英文简历，手动翻译费时且容易出错
- **缺乏专业自我介绍**: 面试时的2分钟口语自我介绍，多数人临时组织语言效果不佳
- **无法离线导出**: 在线简历工具导出受限，需要 Word/PDF 离线文件用于邮件投递

## ② 完整执行业务流水线
```
01_jd_parser JD 数据 (关键词 + 权重)
   +
02_resume_parser 简历素材 (教育/工作/项目/技能)
   +
03_info_enquiry 补充信息 (追问回答)
   ↓
TargetResumeGenerator.generate()
  ├── JD关键词匹配排序: 经历按匹配度从高到低排列
  ├── 五模块简历构建:
  │     ├── 基本信息 (姓名/电话/邮箱/城市/目标岗位)
  │     ├── 个人概述 (技能×JD 匹配摘要)
  │     ├── 工作经历 (标注JD匹配度高/中/低)
  │     ├── 教育经历
  │     ├── 技能证书 (JD匹配 vs JD缺失)
  │     └── JD缺口分析
  ├── 中英自我介绍 (350字/110-130秒)
  └── Word + PDF 离线导出
```

## ③ 防幻觉机制
- **仅重组不改写**: 只调顺序、优化表述，不新增经历/项目/KPI
- **无量化不编造**: 原文没有提升30%、增长200万等数据 → 保持空白，不填虚假数字
- **技能匹配如实标注**: JD matched / JD missing 如实列出，不美化

## ④ 与其他模块的数据互通逻辑
- **输入来自 01_jd_parser**: JD 关键词 → 经历排序权重
- **输入来自 02_resume_parser**: 简历结构化数据 → 唯一素材源
- **输入来自 03_info_enquiry_agent**: 追问补充信息 → 合并到素材
- **输出给 web_ui**: 简历数据 → 双端预览渲染
- **输出 word_pdf_exporter**: 结构化数据 → docx + pdf 文件
