# 03_info_enquiry_agent — AI 信息补充追问 Agent

## ① 模块解决的核心痛点
- **JD 与简历信息不对称**: 岗位要求 Python + Docker，但简历中只提到 Python，HR 无法判断候选人是否具备 Docker 能力
- **简历缺少量化数据**: 很多人写的简历缺乏具体成果数字，AI 能识别并引导补充
- **项目经历缺失**: 初级候选人简历普遍缺少项目细节，需引导补充

## ② 完整执行业务流水线
```
01_jd_parser 输出 JD 技能关键词
   +
02_resume_parser 输出简历技能清单
   ↓
InfoEnquiryAgent.generate_questions()
   ├── 技能缺口检测: JD 技能 ∩ 简历技能 = 缺失项 → 逐个生成追问
   ├── 量化数据检测: 工作经历中搜索 %/数字/增长率 → 缺失 → 追问
   ├── 项目细节检测: 简历中无项目模块 → 追问
   └── 年限缺口检测: JD 要求 5 年 vs 简历展示 2 段 → 追问
   ↓
用户回答后 → process_answer() → 回写素材库
```

## ③ 防幻觉机制
- **严格基于 JD vs 简历差异**: 只提问 JD 明确要求但简历缺失的内容，不发散
- **不回写虚构内容**: 用户回答原样存入素材库，标记 source='问卷补充' 区分原始简历
- **分轮轻量提问**: 每轮最多 1-2 个问题，不堆砌、不压榨

## ④ 与其他模块的数据互通逻辑
- **输入来自 01_jd_parser**: JD 技能关键词 → 对比基准
- **输入来自 02_resume_parser**: 简历已提取数据 → 已有信息集合
- **输出给 04_target_resume_generator**: 用户补充的问答内容 → 并入简历生成素材
- **输出给素材库 (MaterialStore)**: 新增技能、QA 记录 → 全局素材读写
