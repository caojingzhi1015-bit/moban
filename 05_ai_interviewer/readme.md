# 05_ai_interviewer — AI 模拟面试官对话系统

## ① 模块解决的核心痛点
- **面试准备不足**: 候选人不知道面试官会问什么，缺乏真实面试演练
- **简历经历被质疑**: 面试官追问项目细节时，候选人容易答不上来或前后矛盾
- **缺乏多角色模拟**: HR面/技术面/压力面/英文面各有不同策略，需要针对训练

## ② 完整执行业务流水线
```
01_jd_parser JD 岗位信息
   +
02_resume_parser 个人简历素材
   ↓
AIInterviewer.init_session()
   ↓
5 阶段结构化面试:
  暖场(0-5min)   → 自我介绍，观察表达结构
  背景核实(5-15)  → 时间线/职级/离职原因逐条核实
  STAR验证(15-30) → 情景→任务→行动→结果深挖
  压力测试(30-40) → 质疑成果、极端情境
  候选人提问(40-45)→ 中性收尾
   ↓
generate_report() → 评估报告: 匹配度/能力表现/STAR质量/风险点
```

## ③ 防幻觉机制
- **严格基于简历**: 面试官系统提示词写入完整简历文本 + JD，只提问已有内容
- **禁止延伸提问**: "你不存在的项目/经历/技能，面试官不会提问"
- **STAR 追问约束**: 追问方向锁定在用户刚回答的内容上，不跳转话题

## ④ 与其他模块的数据互通逻辑
- **输入来自 01_jd_parser**: JD 岗位 → 面试官据此核对候选人匹配度
- **输入来自 02_resume_parser**: 简历数据 → 面试官唯一提问依据
- **输入来自 common/multi_model_gateway**: 对话 AI 调用
- **输出评估报告**: 供候选人自我改进参考
